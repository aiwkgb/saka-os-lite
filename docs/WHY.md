# Why SAKA OS Lite Exists

## English summary

SAKA OS Lite exists because many solo operators and non-technical founders do not fail from lack of ideas. They fail because their priorities, follow-ups, content, opportunities, and project state are scattered.

This repo offers a small public-safe operating template: JSON data, Python scripts, and a static dashboard.

It is intentionally simple so low-resource builders can understand it, copy it, and adapt it before spending money on complex SaaS tools or AI automation.

## Ringkasan Bahasa Indonesia

SAKA OS Lite dibuat karena banyak solo operator, freelancer, dan founder non-teknis bukan kekurangan ide. Mereka sering kewalahan karena:

- project tersebar,
- prioritas tidak jelas,
- follow-up lupa,
- content tidak punya pipeline,
- peluang/freebie tidak tercatat,
- ketika berhenti kerja, sulit mulai lagi.

SAKA OS Lite memberi struktur kecil yang bisa dipakai sebelum masuk ke automation berat.

## Core idea

> Simple source-of-truth first. AI automation later.

Before using AI agents, a solo operator needs a place to store:

- active projects,
- next actions,
- content state,
- opportunity watchlist,
- daily brief/log rhythm.

## Design principles

1. **Demo-data only** — safe for public GitHub.
2. **File-first** — JSON, Markdown, Python, static dashboard.
3. **Low-cost** — no paid API required.
4. **Beginner-readable** — no complex backend required.
5. **Agent-ready later** — clear data makes future AI workflows easier.

## What this is not

SAKA OS Lite is not:

- a full SaaS product,
- a private data vault,
- a replacement for CRM/accounting/legal tools,
- an autonomous agent by itself,
- a place to publish sensitive data.

## Who should use it

Use it if you are:

- a solo operator,
- a freelancer,
- a small business operator,
- a non-technical founder,
- an AI-assisted builder who needs structure before automation.

## Success condition

This template is useful if it helps someone answer in under 2 minutes:

1. What should I focus on today?
2. Which project needs action?
3. What content or opportunity is waiting?
4. What did I finish?
5. Where do I resume tomorrow?
