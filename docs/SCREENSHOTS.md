# Screenshot Guide

Screenshots help people understand the project quickly, but they can leak private data.

## Safe screenshot rules

Only capture screenshots that contain:

- demo data,
- fictional names,
- placeholder metrics,
- public-safe UI,
- no private browser tabs,
- no terminal showing tokens or paths to secrets.

## Do not capture

- real client names,
- real leads,
- invoices,
- ad accounts,
- bank/e-wallet/finance data,
- private chats,
- API keys/tokens/passwords,
- private dashboard from your real operation.

## Recommended screenshots

1. Dashboard overview with demo data.
2. Terminal output of `python3 scripts/generate_brief.py`.
3. Folder structure of the repo.
4. Example JSON file with fictional data.

## Suggested filenames

```text
assets/screenshots/dashboard-demo.png
assets/screenshots/daily-brief-terminal.png
assets/screenshots/project-radar-json.png
```

## Before publishing

Check every screenshot manually.

If unsure, do not publish it.
